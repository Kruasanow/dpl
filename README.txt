    Атака "перехват пароля" (Password sniffing attack) - это атака, при которой злоумышленник перехватывает сетевой трафик и захватывает пароли, передаваемые по протоколу FTP в открытом виде, без шифрования.

    Атака "подделка" (Spoofing attack) - злоумышленник создает поддельный FTP-сервер или изменяет DNS-записи, чтобы пользователи перенаправлялись на поддельный сервер. Злоумышленник может затем перехватывать пароли, отправленные пользователями на поддельный сервер.

    Атака "отказ в обслуживании" (Denial of Service (DoS) attack) - злоумышленник отправляет большое количество запросов на FTP-сервер, чтобы затопить его и сделать его недоступным для других пользователей.

    Атака "взлом сервера" (FTP server hacking) - злоумышленник пытается найти уязвимости в FTP-сервере, чтобы получить несанкционированный доступ к файлам на сервере.

    Атака "загрузка вредоносного ПО" (FTP malware injection) - злоумышленник загружает вредоносное ПО на FTP-сервер, чтобы заразить компьютеры пользователей, которые загружают файлы с сервера.

    Атака "злоупотребление привилегиями" (FTP privilege escalation attack) - злоумышленник пытается получить расширенные привилегии на FTP-сервере, чтобы получить доступ к файлам, на которые у него нет права доступа.
